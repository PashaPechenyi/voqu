# US-005-1.1: Public Layout & Routing

## Story

**As a** developer
**I want to** have a layout wrapper and routing configuration
**So that** all public pages share consistent header/footer and navigation works correctly

---

## Description

Create the PublicLayout component that wraps all public pages with header and footer, and set up the routing configuration using React Router. Also configure the app entry point with all necessary providers.

---

## Acceptance Criteria

### PublicLayout Component
- [ ] Create component at `apps/web/src/components/layout/PublicLayout.tsx`
- [ ] Render PublicHeader at the top
- [ ] Render Outlet (from react-router) for page content in the middle
- [ ] Render PublicFooter at the bottom
- [ ] Use flexbox column layout with minHeight 100vh
- [ ] Main content area should flex: 1 to push footer down

### Routes Configuration
- [ ] Create routes file at `apps/web/src/routes/index.tsx`
- [ ] Use createBrowserRouter from react-router-dom
- [ ] Configure PublicLayout as parent element
- [ ] Add route "/" for LandingPage
- [ ] Add route "/about" for AboutPage
- [ ] Export router for use in RouterProvider

### App Entry Point
- [ ] Update `apps/web/src/main.tsx`
- [ ] Wrap app in React.StrictMode
- [ ] Wrap in HelmetProvider (for SEO)
- [ ] Wrap in ThemeProvider with theme
- [ ] Include CssBaseline for consistent styles
- [ ] Use RouterProvider with configured router

---

## Technical Notes

**Libraries:**
- `react-router-dom` - createBrowserRouter, RouterProvider, Outlet
- `@mui/material` - Box, ThemeProvider, CssBaseline
- `react-helmet-async` - HelmetProvider

**File locations:**
- `apps/web/src/components/layout/PublicLayout.tsx`
- `apps/web/src/routes/index.tsx`
- `apps/web/src/main.tsx`

**Provider nesting order (outer to inner):**
1. React.StrictMode
2. HelmetProvider
3. ThemeProvider
4. CssBaseline
5. RouterProvider

---

## UI/UX Requirements

- Page transitions should be instant (no loading indicators for client-side nav)
- Content area scrolls while header stays sticky
- Footer stays at bottom even with minimal content
- Consistent spacing and appearance across all pages
